  • 为什么加入透镜？

    为了更好的成像效果。

  • 加入透镜对成像的影响及原因？

    产生畸变，原因：一是透镜自身形状对光纤传播的影响（径向畸变），二是在机械组装过程中，透镜和成像平面不可能完全平行。

6.1 namespace

命名空间：可以作为附加信息来区分不同库中同名的函数、类、变量等。使用命名空间就定义了上下文。本质上，命名空间定义了一个范围。

:: 作用域解析运算符，用于辨识和标出属于哪个作用域上下文，特别是属于哪个命名空间

6.2 编译

使用 g++

使用编译器将一个文本文件编译成可执行程序

例如：slambook/ch2/helloSALM.cpp

可以使用 gedit 或者 vim 编辑以上代码，然后使用 g++ 将以上代码编译成一个可执行文件。

执行以上命令，检查当前目录会发现多了一个 a.out 文件，它具有可执行性（终端里显示为绿色的可执行文件）。

使用 cmake

在实际的工程里会有许多的文件夹和源文件，通常一个小型 c++ 项目会包含十几个类，各类间还存在着复杂的依赖关系，其中一部分要编译成可执行文件，一部分要编译成库文件，这时输入的编译命令会很长，整个编译过程会变得异常繁琐。因而采用 cmake进行自动编译。

cmake：使用 cmake 命令生成一个 makefile 文件，然后 make 会根据 makefile 文件内容编译整个工程。

例子：slambook/ch2/CMakeLists.txt

CMakeLists.txt 文件告诉 cmake 我们要对该目录下的文件做什么，CMakeLists.txt 文件内容要遵循 cmake 语法。

cmake 会输出一些编译器等信息并在当前目录下生成一些中间文件，其中最重要的是 MakeFile，它是自动生成的。
